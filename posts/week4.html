Week 1 - Flex vs Grid display

One of the most frustrating aspects of designing a webpage can be struggling with the positioning and sizing of the content. When I first used flex display, I immediately saw the benefits. Now that I have been confronted with grid display as well, I am curious to see how these display systems may overlap, and in which situations they are most useful

The above show that very similar methods of display can be achieved using both methods. What the above examples do not show, however, was how simple and elegantly each type of display was achieved. With flex, applying row and column layout was extremely simple, and spacing the content in various ways was simple enough to include an animation, refresh to view again. Grid layout acheived the same layouts, but visibly handled the elements differently.

Notice how each element is sized. In flex style, objects fill only the space they need. In grid, the object seems to fill the entire grid cell. While particular styling could eventually make either style look identical to the other, knowing what to expect as typical behavior from these display schemes can surely assist the design process.

It seems as though flex display is more appropriate for organizing content that needs to flow, or have a more organic nature. While it wasn't used in the examples above, flex has the capability of wrapping content in its container as elements begin to overflow. This is useful for when the page is resized, and when it is unknown how large all the elements in the container will be. Grid layout seems appropriate for displaying content in a fixed way. Flex is grid-like, however Grid layout uses fixed columns and grids to organize the content. Grids can be useful for displaying content with particular alignment, there should be no use for wrapping in a grid, and flex display gives no control or guarantee of the number of rows or columns.

A larger list of comparisons between flex and grid display can be found here, and among them can be seen many more similarities behind each style, and how they can slightly differ. What I found most interesting was the idea given by the article of one-dimensional vs. two-dimensional layout. This may be a better way to contrast the two styles. Flex style appears two-dimensional, but in reality is only a dynamically fitted representation of a one-dimensional construct.

Ultimately, there are several display options available. A list can be found here. Flex and grid displaya are only two solutions that cannot possibly be applied to every design problem. Perhaps I will explore the other options to see if any would improve my own content.
