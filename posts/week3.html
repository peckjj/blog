                    #boo {
                        animation: fadeInAnimation ease 3s;
                        animation-iteration-count: 1;
                        animation-fill-mode: forwards;
                    }

                    @keyframes fadeInAnimation {
                        0% {
                            opacity: 0;
                        }
                        75% {
                            opacity: 0;
                        }
                        95% {
                            opacity: 1;
                        }
                        100% {
                            opacity: 0;
                        }
                    }
                </style>

This was a fun tool to try, and the format is simple enough to try larger animations. The animation property is documented here, and more examples can be found here.

The animation property is very similar to transition in that you set attributes of an element to be tracked. The animation property allows custom animation to be defined, and there is no limit to the number of attributes that can be controlled at once in any given animation.

The blog site already contains an early version of an animation. The cards that contain the main content slide and fade in from the side of the screen. This is controlled with the animation property.

animation vs transition

Due to the power of animation, One may ask why transition would ever be used. This site explains transitions and animations fairly plainly. It also argues that transitions may be more appropriate for smaller, simpler animations. They are simpler to manage, as multiple transitions produce far less clutter than multiple animations. For more complex behavior, animations can be used to produce infinite behavior, or advanced control over transition intervals.

Moving forward with the website, interactive design will permeate style additions to get closer to an interface that is properly engaging to the user.
